  <name>Task 1: Commit detail cache and popup panel units</name>
  <files>src/DX.Blame.CommitDetail.pas, src/DX.Blame.Popup.pas</files>
  <action>
Create two new units following project header conventions (XML doc, MIT license, Copyright 2026 Olaf Monien). UTF-8 BOM, CRLF line endings.

**DX.Blame.CommitDetail.pas:**
- Define `TCommitDetail` record: `FullMessage: string`, `FileDiff: string`, `FullDiff: string`, `Fetched: Boolean`.
- Define `TCommitDetailCache` class with `TCriticalSection` + `TObjectDictionary<string, TCommitDetail>` (keyed by 40-char commit hash). The dictionary should store pointers or use a wrapper class since TObjectDictionary needs class values -- alternatively use `TDictionary<string, TCommitDetail>` since TCommitDetail is a record (no ownership needed).
- Methods: `function TryGet(const AHash: string; out ADetail: TCommitDetail): Boolean`, `procedure Store(const AHash: string; const ADetail: TCommitDetail)`, `procedure Clear`.
- `procedure FetchCommitDetailAsync(const ACommitHash, ARepoRoot, ARelativeFilePath: string; AOnComplete: TProc<TCommitDetail>)` -- standalone procedure that spawns `TThread.CreateAnonymousThread`:
  - Uses `TGitProcess` (from DX.Blame.Git.Process) to run `git log -1 --format=%B <hash>` for full message.
  - Runs `git show <hash> -- <relpath>` for file-specific diff.
  - Runs `git show <hash>` for full-commit diff.
  - Delivers result via `TThread.Queue(nil, ...)` to main thread.
  - **Important:** Do NOT use `TProc<TCommitDetail>` as a callback parameter directly due to Delphi generic anonymous method type issues (see Phase 2 decision). Instead, use a dedicated thread class `TCommitDetailThread` following the `TBlameThread` pattern from DX.Blame.Engine.pas -- store the result in thread fields and deliver via `TThread.Queue` with a non-generic callback approach. Alternatively, test if `TProc<TCommitDetail>` works in this context since it is a concrete specialization, not a method reference stored in a generic container.
- Singleton `function CommitDetailCache: TCommitDetailCache` with lazy init and finalization cleanup.

**DX.Blame.Popup.pas:**
- `TDXBlamePopup = class(TCustomForm)` -- borderless popup panel, created programmatically (no DFM).
- `constructor CreateNew(AOwner: TComponent)` -- sets `BorderStyle := bsNone`, adds WS_POPUP | WS_BORDER via `CreateParams` override, WS_EX_TOOLWINDOW to avoid taskbar button.
- `procedure CMDeactivate(var Message: TMessage); message CM_DEACTIVATE;` -- calls `Hide` for click-outside dismissal.
- Escape key handling: override `KeyDown` or use `CMDialogKey` to hide on VK_ESCAPE.
- Layout (all created in code):
  - `FHashLabel: TLabel` -- shows 7-char short hash, styled as clickable (hand cursor, underline). On click: copy full 40-char SHA to clipboard via `Vcl.Clipbrd.Clipboard.AsText`, change label text to "Copied!" for 1.5 seconds using a `TTimer` (owned by the form). Use a handler method on the form class for the timer OnTimer event (not anonymous method -- Delphi cannot assign anon methods to TNotifyEvent).
  - `FAuthorLabel: TLabel` -- "Author Name <email>"
  - `FDateLabel: TLabel` -- full absolute date/time via `FormatDateTime('yyyy-mm-dd hh:nn:ss', ALineInfo.AuthorTime)`
  - `FMessageMemo: TMemo` -- full multi-line commit message, read-only, no border or minimal border, 4-6 visible lines
  - `FShowDiffButton: TButton` -- "Show Diff" caption, anchored at bottom. OnClick handler stored as a `TNotifyEvent` field `FOnShowDiffClick` set from outside (the renderer/registration will wire this to open the diff dialog in Plan 02). For now, create the button but leave the handler unassigned.
  - `FLoadingLabel: TLabel` -- "Loading..." text shown while async fetch is in progress, hidden when data arrives.
- Theme adaptation: On show, detect dark/light theme using the same approach as `DeriveAnnotationColor` (read IDE editor background color luminance). Set form Color, label Font.Color, memo Color accordingly. Use clWindow/clWindowText for light, dark gray (#2D2D2D) / light gray (#D4D4D4) for dark theme.
- `procedure ShowForCommit(const ALineInfo: TBlameLineInfo; const AScreenPos: TPoint; const ARepoRoot, ARelativeFilePath: string)`:
  - If `ALineInfo.IsUncommitted`: show simplified panel with just "Not committed yet" message, no hash, no diff button.
  - Otherwise: populate hash, author, date from ALineInfo fields immediately. Show loading label. Call `FetchCommitDetailAsync` to get full message. On callback: hide loading label, set FMessageMemo.Text to FullMessage. Store the fetched detail in CommitDetailCache.
  - Before fetching: check `CommitDetailCache.TryGet` -- if already cached, populate immediately without async fetch.
  - Position popup at AScreenPos (screen coordinates), adjust to keep within screen bounds.
- `procedure UpdateContent(const ALineInfo: TBlameLineInfo; const ARepoRoot, ARelativeFilePath: string)` -- called when clicking a different annotation while popup is visible. Updates all fields in-place without hiding/showing (no flicker).
- Form width ~400, height auto-sized to content (min ~200, max ~400).

Both units use the project naming conventions (L-prefix locals, F-prefix fields, A-prefix params).

**DX.Blame.Renderer.pas -- click detection and popup management:**
- Add a field `FAnnotationRects: TDictionary<Integer, Integer>` (maps visible screen row Y-range to annotation start X). Alternatively, store a single `FMinAnnotationX: Integer` if all annotation X values are similar, but per-line is more accurate. Best approach: store `FAnnotationXByRow: TDictionary<Integer, Integer>` mapping `Rect.Top` (the paint rect top) to `LAnnotationX` computed in PaintLine. Clear this dictionary in `BeginPaint`.
- In `PaintLine`, after computing `LAnnotationX` and before drawing: store `FAnnotationRects[Rect.Top] := LAnnotationX`.
- Add a field `FPopup: TDXBlamePopup` (lazy-created on first click).
- Add a field `FCellHeight: Integer` updated from `Context.CellSize.cy` in PaintLine.
- Implement the `EditorMouseDown` (INTACodeEditorEvents370 overload with `var Handled`):
  - Only handle `mbLeft`, only when `BlameSettings.Enabled`.
  - Look up the annotation X for the clicked row: find the entry in `FAnnotationRects` where `Key <= Y < Key + FCellHeight`. If `X >= that annotation X value`, the click is in the annotation area.
  - Determine the logical line: compute `LVisibleRow := (Y - FAnnotationRects.Keys.First) div FCellHeight` + top visible line. Or better: iterate `FAnnotationRects` to find the matching row rect, then derive line from `Context` info. Since we don't have Context in MouseDown, store a parallel `FLineByRow: TDictionary<Integer, Integer>` mapping `Rect.Top` to logical line number during PaintLine.
  - Get blame data: `BlameEngine.Cache.TryGet(filename, LBlameData)`, then `LBlameData.Lines[LLineIndex]`.
  - Get screen position: `LScreenPos := Editor.ClientToScreen(Point(X, Y))`.
  - Get repo root from `BlameEngine.RepoRoot`.
  - Compute relative file path: `AFileName` relative to `BlameEngine.RepoRoot` using `ExtractRelativePath` or string manipulation.
  - If popup is nil, create it: `FPopup := TDXBlamePopup.CreateNew(nil)`.
  - If popup is already visible, call `FPopup.UpdateContent(...)`. Otherwise call `FPopup.ShowForCommit(...)`.
  - Set `Handled := True` to prevent IDE from processing the click.
- Add uses: `DX.Blame.Popup, DX.Blame.CommitDetail, System.Generics.Collections`.
- In the destructor or a cleanup method: `FreeAndNil(FPopup)`, `FreeAndNil(FAnnotationRects)`, `FreeAndNil(FLineByRow)`.
- **Important:** TDXBlameRenderer is reference-counted (TNotifierObject). It does not have a destructor called in the normal way. Add cleanup in `EndPaint` is wrong. Instead, expose a module-level `procedure CleanupPopup` that frees the popup, called from Registration finalization. Or store the popup reference in a unit-level var in Renderer.pas and free in finalization.
- Hide the popup when editor context changes: in `EditorScrolled`, hide popup if visible. Also hide in `BeginPaint` if the editor TWinControl changed (different tab).
