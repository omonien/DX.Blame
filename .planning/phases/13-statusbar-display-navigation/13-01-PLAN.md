**3. Add GOnCaretMoved to DX.Blame.Renderer.pas:**
- In the interface section, add:
  ```pascal
  var
    GOnCaretMoved: procedure(const AFileName: string; ALine: Integer) of object;
  ```
  Note: Use `of object` method pointer so TDXBlameStatusbar.UpdateForLine can be assigned directly. Alternatively, use a plain procedure variable `procedure(const AFileName: string; ALine: Integer)` — check which fits the TDXBlameStatusbar method signature.

  Actually, per research pattern (matching `OnBlameToggled: TProc` in KeyBinding), use a simple procedure variable:
  ```pascal
  var
    GOnCaretMoved: procedure(const AFileName: string; ALine: Integer);
  ```
- In `EditorSetCaretPos`, after `InvalidateAllEditors;`, add:
  ```pascal
  if Assigned(GOnCaretMoved) and (FCurrentFileName <> '') then
    GOnCaretMoved(FCurrentFileName, FCurrentLine);
  ```
  IMPORTANT: `FCurrentLine` is set during PaintLine (from EditView.CursorPos.Line), NOT from EditorSetCaretPos Y (which is view-relative). On first call, FCurrentLine may be 0 or stale. The statusbar UpdateForLine must handle line=0 gracefully (show empty text). This is acceptable because PaintLine fires immediately after EditorSetCaretPos due to InvalidateAllEditors, and the NEXT caret move will have the correct FCurrentLine from the previous paint cycle. The one-paint-cycle delay is imperceptible.

**4. Create src/DX.Blame.Statusbar.pas:**
- Standard unit header (MIT license, copyright 2026 Olaf Monien)
- Class `TDXBlameStatusbar = class(TComponent)` — TComponent base required for FreeNotification
- Private fields:
  - `FPanel: TStatusPanel` — the injected panel (nil when detached or statusbar destroyed)
  - `FStatusBar: TStatusBar` — the host statusbar (nil when detached or destroyed)
  - `FPanelIndex: Integer` — stored panel index for cleanup
  - `FLineInfo: TBlameLineInfo` — last-displayed line info for click handler
  - `FHasBlameData: Boolean` — whether FLineInfo contains valid data
  - `FFileName: string` — last-displayed file name for popup relative path
  - `FOldOnMouseDown: TMouseEvent` — original OnMouseDown handler to chain
- Protected: override `Notification(AComponent: TComponent; Operation: TOperation)` — when `(Operation = opRemove) and (AComponent = FStatusBar)`, nil out FPanel, FStatusBar, set FPanelIndex := -1. This handles IDE destroying the edit window form.
- Constructor: `Create(AOwner: TComponent)` — call inherited, set `FPanelIndex := -1`
- Destructor: call `DetachFromStatusBar` then inherited
- Public methods:
  - `procedure AttachToStatusBar(AStatusBar: TStatusBar)` — detach if already attached, store FStatusBar, call `AStatusBar.FreeNotification(Self)`, create panel via `FStatusBar.Panels.Add`, set Width=300, Style=psText, Text='', store FPanelIndex. Save and chain existing OnMouseDown: `FOldOnMouseDown := FStatusBar.OnMouseDown; FStatusBar.OnMouseDown := HandleStatusBarMouseDown;`
  - `procedure DetachFromStatusBar` — if FStatusBar <> nil and FPanelIndex valid, delete the panel. Restore `FStatusBar.OnMouseDown := FOldOnMouseDown`. Call `FStatusBar.RemoveFreeNotification(Self)`. Nil out all references.
  - `procedure UpdateForLine(const AFileName: string; ALine: Integer)` — guard: if FPanel = nil or FStatusBar = nil, exit. If not BlameSettings.ShowStatusbar, set FPanel.Text := '' and exit. Call `BlameEngine.Cache.TryGet(AFileName, LData)`. If line valid, format with `FormatBlameAnnotation(LData.Lines[ALine-1], BlameSettings)`, store in FLineInfo, set FHasBlameData := True, store AFileName in FFileName. Set `FPanel.Text := LText`. If no data or line out of range, clear panel text and set FHasBlameData := False.
  - `procedure HandleStatusBarMouseDown(Sender: TObject; Button: TMouseButton; Shift: TShiftState; X, Y: Integer)` — this is the OnMouseDown handler. First check if click is within FPanel's bounds: iterate FStatusBar.Panels to compute left offset of FPanel, check if X falls within [panelLeft, panelLeft + FPanel.Width]. If hit and FHasBlameData and Button=mbLeft: compute screen position via `FStatusBar.ClientToScreen(Point(X, 0))`, call popup. For popup: import DX.Blame.Popup in implementation uses, access GPopup. Wait — GPopup is a unit-level var in Renderer.pas, not exported. Instead, create the popup locally or export a procedure. Better approach per research: import DX.Blame.Popup and create a local popup instance, OR call a public procedure. Actually, the research says to reuse GPopup via `GPopup.ShowForCommit`. GPopup is declared in Renderer.pas implementation section — not accessible. Two options: (a) export a ShowPopupForCommit wrapper from Renderer.pas, or (b) create a TDXBlamePopup instance in the statusbar unit. Option (b) is cleaner — the statusbar has its own popup lifecycle. Create `FPopup: TDXBlamePopup` lazily. On click: compute relative path from FFileName and BlameEngine.RepoRoot, then call `FPopup.ShowForCommit(FLineInfo, LScreenPos, BlameEngine.RepoRoot, LRelPath)`. Use the same relative path computation as Renderer.pas (ExtractRelativePath + forward slashes). After calling ShowForCommit, chain to FOldOnMouseDown if assigned (for other panel clicks).
  - If the click is NOT on our panel, chain to `FOldOnMouseDown`

- Interface uses: `System.Classes, Vcl.ComCtrls` (for TStatusBar, TStatusPanel)
- Implementation uses: `System.SysUtils, System.Types, Vcl.Controls, Vcl.Forms, DX.Blame.Settings, DX.Blame.Formatter, DX.Blame.Engine, DX.Blame.VCS.Types, DX.Blame.Cache, DX.Blame.Popup`

**6. Wire statusbar in DX.Blame.Registration.pas:**
- Add `DX.Blame.Statusbar` to implementation uses clause
- Add unit-level var: `GStatusbar: TDXBlameStatusbar = nil;`
- In `Register` procedure, after `AttachContextMenu` call:
  ```pascal
  // Create and attach statusbar blame panel
  GStatusbar := TDXBlameStatusbar.Create(nil);
  var LNTAServices: INTAEditorServices;
  if Supports(BorlandIDEServices, INTAEditorServices, LNTAServices) then
  begin
    var LEditWindow := LNTAServices.TopEditWindow;
    if (LEditWindow <> nil) and (LEditWindow.StatusBar <> nil) then
      GStatusbar.AttachToStatusBar(LEditWindow.StatusBar);
  end;

  // Wire caret-moved callback to statusbar update
  DX.Blame.Renderer.GOnCaretMoved := GStatusbar.UpdateForLine;
  ```
  NOTE: `GOnCaretMoved` is a plain procedure variable, but `UpdateForLine` is a method — this won't compile with a plain `procedure(...)` variable. Solution: declare `GOnCaretMoved` as `TCaretMovedEvent = procedure(const AFileName: string; ALine: Integer) of object` and use that type. OR use a standalone wrapper procedure in Registration.pas that calls GStatusbar.UpdateForLine. The wrapper approach is simpler and avoids changing Renderer's interface type:
  ```pascal
  procedure OnCaretMovedHandler(const AFileName: string; ALine: Integer);
  begin
    if GStatusbar <> nil then
      GStatusbar.UpdateForLine(AFileName, ALine);
  end;
  ```
  Then: `DX.Blame.Renderer.GOnCaretMoved := OnCaretMovedHandler;`

- In finalization, insert between CleanupPopup (step 3) and UnregisterRenderer (step 4):
  ```pascal
  // 3.5 Clean up statusbar panel before stopping renderer
  DX.Blame.Renderer.GOnCaretMoved := nil;
  if GStatusbar <> nil then
  begin
    GStatusbar.DetachFromStatusBar;
    FreeAndNil(GStatusbar);
  end;
  ```
