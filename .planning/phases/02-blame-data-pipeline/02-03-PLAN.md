**TBlameThread (private class, defined in implementation section):**
Subclass of TThread. Executes git blame in background.
- Fields: `FFileName: string`, `FRepoRoot: string`, `FGitPath: string`, `FProcessHandle: THandle`, `FOnComplete: TProc<string, TBlameData>`, `FOnError: TProc<string, string>` (filename, error message)
- `procedure Execute; override`:
  1. Create TGitProcess with FGitPath and FRepoRoot
  2. Compute relative path: `ExtractRelativePath(IncludeTrailingPathDelimiter(FRepoRoot), FFileName)`. Replace backslashes with forward slashes for git.
  3. Execute: `LExitCode := LProcess.ExecuteAsync('blame --line-porcelain -- "' + LRelPath + '"', LOutput, FProcessHandle)`
  4. If Terminated, exit without callback
  5. If LExitCode = 0: parse with `ParseBlameOutput(LOutput, LLines)`, create TBlameData, set Lines/FileName/Timestamp, call `TThread.Queue(nil, procedure begin FOnComplete(FFileName, LData); end)`
  6. If LExitCode <> 0: call FOnError via TThread.Queue
- `procedure Cancel`:
  1. Terminate (set Terminated flag)
  2. `TGitProcess.CancelProcess(FProcessHandle)` to kill the git process

- FreeOnTerminate := True (thread cleans up itself)

**TBlameEngine (public class):**
```pascal
TBlameEngine = class
private
  FCache: TBlameCache;
  FGitPath: string;
  FRepoRoot: string;
  FGitAvailable: Boolean;
  FGitNotified: Boolean; // one-time "git not found" notification flag
  FActiveThreads: TDictionary<string, TBlameThread>; // key = lowercase filename
  FDebounceTimers: TDictionary<string, TTimer>; // key = lowercase filename
  FLock: TCriticalSection; // guards FActiveThreads and FDebounceTimers

  procedure HandleBlameComplete(const AFileName: string; AData: TBlameData);
  procedure HandleBlameError(const AFileName: string; const AError: string);
  procedure DoRequestBlame(const AFileName: string);
  procedure LogToIDE(const AMessage: string);
  function IsSourceFile(const AFileName: string): Boolean;
public
  constructor Create;
  destructor Destroy; override;

  procedure Initialize(const AProjectPath: string);
  procedure RequestBlame(const AFileName: string);
  procedure RequestBlameDebounced(const AFileName: string);
  procedure CancelAndRemove(const AFileName: string);
  procedure OnProjectSwitch(const ANewProjectPath: string);

  property Cache: TBlameCache read FCache;
  property GitAvailable: Boolean read FGitAvailable;
  property RepoRoot: string read FRepoRoot;
end;
```

**Initialize(AProjectPath):**
1. `FGitPath := FindGitExecutable`. If empty: set FGitAvailable := False, log one-time message via `LogToIDE('DX.Blame: git not found on PATH or common locations. Blame features disabled.')`, exit.
2. `FRepoRoot := FindGitRepoRoot(AProjectPath)`. If empty: set FGitAvailable := False (not a git repo, silent -- per user decision for untracked/non-repo), exit.
3. Set FGitAvailable := True.

**RequestBlame(AFileName):**
1. If not FGitAvailable, exit silently.
2. If not IsSourceFile(AFileName), exit (skip .dfm, .res, .dcu etc. -- only blame .pas, .dpr, .dpk, .inc files).
3. Cancel any existing thread for this file (via CancelAndRemove logic for the thread only, keep cache).
4. Create TBlameThread with FGitPath, FRepoRoot, AFileName, HandleBlameComplete, HandleBlameError. FreeOnTerminate := True.
5. Store thread reference in FActiveThreads (lowercase key).
6. Start thread.

**RequestBlameDebounced(AFileName):**
1. If not FGitAvailable, exit.
2. If a debounce timer exists for this file, reset its interval (disable then enable).
3. Otherwise create a new TTimer (Interval := cDefaultDebounceMs, OnTimer fires DoRequestBlame, Enabled := True).
4. Store timer in FDebounceTimers.

**DoRequestBlame (timer callback):**
1. Disable and free the timer, remove from FDebounceTimers.
2. Invalidate cache for the file.
3. Call RequestBlame.

**CancelAndRemove(AFileName):**
1. Lock FLock.
2. If thread exists in FActiveThreads: call thread.Cancel, remove from dictionary (do NOT free -- FreeOnTerminate handles it).
3. If debounce timer exists: disable, free, remove from FDebounceTimers.
4. Unlock.
5. `FCache.Invalidate(AFileName)`.

**OnProjectSwitch(ANewProjectPath):**
1. Cancel all active threads (iterate FActiveThreads, cancel each).
2. Clear FActiveThreads.
3. Disable and free all debounce timers, clear FDebounceTimers.
4. FCache.Clear.
5. ClearDiscoveryCache.
6. Initialize(ANewProjectPath).

**HandleBlameComplete:**
1. Remove thread from FActiveThreads (lock).
2. Store data in cache: `FCache.Store(AFileName, AData)`.
3. (Phase 3 will add: notify UI to repaint.)

**HandleBlameError:**
1. Remove thread from FActiveThreads (lock).
2. Check if this is first failure for this file (use a retry set/flag).
3. If first failure: schedule retry after cDefaultRetryDelayMs using a TTimer. On retry, call RequestBlame.
4. If retry also failed: `LogToIDE('DX.Blame: git blame failed for ' + AFileName + ': ' + AError)`.

**LogToIDE:**
Use `IOTAMessageServices.AddTitleMessage` to write to the IDE Messages window. Acquire service via `Supports(BorlandIDEServices, IOTAMessageServices, LMsgServices)`.

**IsSourceFile:**
Check file extension: `.pas`, `.dpr`, `.dpk`, `.inc` return True. Everything else False.

**Singleton access:**
Declare a unit-level function `function BlameEngine: TBlameEngine` that creates the instance on first call (lazy initialization). The instance is freed in the finalization section.

Add unit to DPK contains clause.

Uses: `DX.Blame.Git.Types`, `DX.Blame.Git.Discovery`, `DX.Blame.Git.Process`, `DX.Blame.Git.Blame`, `DX.Blame.Cache`, `System.Classes`, `System.SysUtils`, `System.SyncObjs`, `System.Generics.Collections`, `Vcl.ExtCtrls`, `ToolsAPI`.

Follow project conventions: UTF-8 BOM, CRLF, XML doc header, F-prefix fields, L-prefix locals, A-prefix params.
