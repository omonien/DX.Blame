Define these types:

1. **TBlameLineInfo** (record): Per-line blame data.
   - `CommitHash: string` (40-char SHA-1, all zeros = uncommitted)
   - `Author: string`
   - `AuthorMail: string`
   - `AuthorTime: TDateTime` (converted from Unix timestamp)
   - `Summary: string` (first line of commit message)
   - `OriginalLine: Integer`
   - `FinalLine: Integer`
   - `IsUncommitted: Boolean`

2. **TBlameData** (class): Container for a file's complete blame results.
   - `Lines: TArray<TBlameLineInfo>` (indexed by final line number, 1-based)
   - `FileName: string` (lowercase normalized full path)
   - `Timestamp: TDateTime` (when blame was executed)
   - Constructor taking filename
   - Destructor (clean)

3. **Constants:**
   - `cUncommittedHash = '0000000000000000000000000000000000000000'`
   - `cNotCommittedAuthor = 'Not committed yet'`
   - `cDefaultRetryDelayMs = 2500` (retry delay for transient git errors)
   - `cDefaultDebounceMs = 500` (save debounce)

Follow project conventions: UTF-8 BOM, CRLF, XML doc header with MIT license, PascalCase, `T` prefix for types, `c` prefix for constants.

  <name>Task 2: Create git discovery unit</name>
  <files>src/DX.Blame.Git.Discovery.pas</files>
  <action>
Create `src/DX.Blame.Git.Discovery.pas` implementing git executable finding and repository root detection. This unit uses only RTL (System.IOUtils, System.SysUtils, Winapi.Windows for GetEnvironmentVariable).

**FindGitExecutable function:**
- Returns the full path to git.exe, or empty string if not found.
- Search order (per user decision):
  1. System PATH: split `GetEnvironmentVariable('PATH')` by `;`, check each dir for `git.exe` using `TFile.Exists(TPath.Combine(LDir, 'git.exe'))`
  2. `C:\Program Files\Git\cmd\git.exe`
  3. `C:\Program Files (x86)\Git\cmd\git.exe`
  4. `C:\Users\<username>\AppData\Local\Programs\Git\cmd\git.exe` (use `GetEnvironmentVariable('LOCALAPPDATA')`)
- Cache the result in a unit-level variable (search once per session).
- Thread-safe: called from main thread only (on project open).

**FindGitRepoRoot function:**
- Takes an absolute path (file or directory) and returns the git repo root, or empty string.
- Walk parent directories looking for `.git` folder using `TDirectory.Exists(TPath.Combine(LDir, '.git'))`.
- Stop at filesystem root (when `TDirectory.GetParent` returns same value).
- Per user decision: after finding `.git` folder, verify with `git rev-parse --show-toplevel` (run synchronously -- fast, <100ms). Use the TGitProcess wrapper from Task 3 if available, or a simple local CreateProcess call for this one synchronous verification.

**ClearDiscoveryCache procedure:**
- Resets cached git path and repo root (called on project switch per user decision).

Follow project conventions: UTF-8 BOM, CRLF, XML doc header, F-prefix for fields, L-prefix for locals, A-prefix for params.

<task type="auto">
  <name>Task 3: Create git process wrapper unit</name>
  <files>src/DX.Blame.Git.Process.pas</files>
  <action>
Create `src/DX.Blame.Git.Process.pas` implementing a safe CreateProcess wrapper for executing git commands with stdout pipe capture and cancellation support.

**TGitProcess class:**
- Constructor: `Create(const AGitPath, AWorkDir: string)`
- `function Execute(const AArgs: string; out AOutput: string): Integer` -- runs git synchronously (for short commands like rev-parse). Returns exit code.
- `function ExecuteAsync(const AArgs: string; out AOutput: string; var AProcessHandle: THandle): Integer` -- same as Execute but exposes the process handle for external cancellation. The handle is valid until the caller closes it.
- `class procedure CancelProcess(var AProcessHandle: THandle)` -- calls `TerminateProcess(AProcessHandle, 1)` then `CloseHandle`. Sets handle to 0.

**CreateProcess setup (per research):**
- `CREATE_NO_WINDOW` flag (no console flash)
- `STARTF_USESTDHANDLES or STARTF_USESHOWWINDOW` for pipe redirection
- `SW_HIDE` for wShowWindow
- Command line: `Format('"%s" %s', [AGitPath, AArgs])`
- Working directory = AWorkDir parameter
- `SetHandleInformation(LReadPipe, HANDLE_FLAG_INHERIT, 0)` to prevent read handle inheritance

**Pipe handling (CRITICAL -- avoid deadlock per research Pitfall 1):**
- Create anonymous pipe with `CreatePipe`
- Close write end of pipe IMMEDIATELY after `CreateProcess` succeeds (before reading)
- Read loop: `ReadFile` into 4096-byte TBytes buffer, append to `TBytesStream`
- Loop until `ReadFile` returns False (broken pipe = process done)
- Decode output as UTF-8: `TEncoding.UTF8.GetString(LStream.Bytes, 0, LStream.Size)`
- After reading: `WaitForSingleObject(LPI.hProcess, 5000)` then `GetExitCodeProcess`

**Handle cleanup (CRITICAL -- per research Pitfall 2):**
- All 5 handles (read pipe, write pipe, process handle, thread handle, security attributes) must be in try/finally blocks.
- Write pipe: close immediately after CreateProcess or in finally if CreateProcess fails.
- Process thread handle: always close in finally.
- Process handle: close in finally for Execute; caller closes for ExecuteAsync.
- Read pipe: close in outer finally.

**Error handling:**
- If `CreateProcess` fails, return -1 and empty output.
- If `CreatePipe` fails, return -1 and empty output.

Do NOT use `TThread.Synchronize` or `TThread.Queue` in this unit -- this is a pure process wrapper. Threading is handled by the Engine unit in Plan 03.

Follow project conventions: UTF-8 BOM, CRLF, XML doc header, proper handle cleanup with try/finally.
