**Renderer changes (DX.Blame.Renderer.pas):**

1. Add `System.Math` to the implementation `uses` clause (needed for `Max` function). Place it after `System.Generics.Collections`.

2. Add a new local variable in PaintLine: `LCaretX: Integer` (in the var section alongside `LAnnotationX`).

3. AFTER the existing `LAnnotationX` calculation (lines 310-311), add:
```pascal
// Caret-anchored: only for caret line, only when setting is active
if (BlameSettings.AnnotationPosition = apCaretColumn) and
   (LLogicalLine = FCurrentLine) and
   (Context.EditView <> nil) and
   (Context.EditView.CursorPos.Col > 0) then
begin
  LCaretX := (Context.EditView.CursorPos.Col - 1) * Context.CellSize.cx +
    Context.LineState.VisibleTextRect.Left;
  LAnnotationX := Max(LCaretX + (Context.CellSize.cx * 3), LAnnotationX);
end;
```

Key design decisions:
- `Max(caretX + padding, endOfLineX)` ensures annotation never moves LEFT of end-of-line (prevents flicker per Pitfall 1 from research).
- `LLogicalLine = FCurrentLine` guard ensures ONLY the caret line gets caret-anchoring in dsAllLines mode (DISP-04).
- `CursorPos.Col > 0` guard handles edge case of invalid column.
- No additional branching for dsAllLines vs dsCurrentLine is needed -- the `FCurrentLine` condition already handles it.
