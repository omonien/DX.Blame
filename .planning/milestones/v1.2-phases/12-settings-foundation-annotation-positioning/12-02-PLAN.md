**Settings (DX.Blame.Settings.pas):**

1. Add private field `FShowInline: Boolean` to TDXBlameSettings (after `FAnnotationPosition`).

2. Add public property:
```pascal
property ShowInline: Boolean read FShowInline write FShowInline;
```

3. In constructor, set `FShowInline := True` BEFORE the `Load` call (after the `FAnnotationPosition := apEndOfLine` line added by Plan 12-01). Default is `True` for backward compatibility -- existing users see no behavior change.

4. In `Load` method, add AFTER the `AnnotationPosition` read (in the 'Display' section):
```pascal
FShowInline := LIni.ReadBool('Display', 'ShowInline', True);
```

5. In `Save` method, add AFTER the `AnnotationPosition` write:
```pascal
LIni.WriteBool('Display', 'ShowInline', FShowInline);
```

**Renderer (DX.Blame.Renderer.pas):**

Add the ShowInline guard in PaintLine IMMEDIATELY after the existing `Enabled` check (after `if not BlameSettings.Enabled then Exit;`):

```pascal
// Skip inline rendering when inline display is disabled
// (blame may still be globally enabled for statusbar display)
if not BlameSettings.ShowInline then
  Exit;
```

IMPORTANT per Pitfall 3 from research: This guard MUST be placed BEFORE any cache lookups or string formatting to avoid wasting work when inline is disabled. The correct position is right after the `Enabled` check (around line 236), before `GCellHeight` assignment.

Anti-pattern: Do NOT create a mode enum like `dmInline/dmStatusbar/dmBoth`. Use two independent Booleans (`ShowInline` now, `ShowStatusbar` in Phase 13). They are orthogonal.
